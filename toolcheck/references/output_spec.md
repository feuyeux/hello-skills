# 输出格式规格

本文档定义了 toolcheck 报告的固定结构，供需要解析输出的下游工具或 agent 参考。

## 报告文件

- 路径：`~/toolcheck/report_MMdd_HHmmss.md`
- 编码：UTF-8
- 格式：Markdown 表格 + 汇总行

## 表格：8 个固定列

| #   | 列名         | 宽度 | 规则                                                 |
| --- | ------------ | ---- | ---------------------------------------------------- |
| 1   | 序号         | 4    | 1-25 递增；重复工具的子行(↳)留空                     |
| 2   | 工具         | 10   | 固定 25 个名称；子行显示 `↳`                       |
| 3   | 状态         | 8    | 仅 5 种值（见下方枚举）                              |
| 4   | 本地版本     | 36   | 版本命令首行截断 36 字符；未安装 → `-`               |
| 5   | 本地安装路径 | 40   | 完整路径截断 40 字符（`...` 前缀）；未安装 → `-`     |
| 6   | 最新版本     | 14   | winget/GitHub API；无源 → `N/A`                      |
| 7   | 操作         | 30   | `保留` / `跳过` / 具体升级命令                       |
| 8   | 备注         | 18   | `未安装` / `仅 macOS` / `超时` / `版本获取失败` / 空 |

## 状态枚举（5 种，互斥）

| 状态值     | 含义       | 触发条件                              |
| ---------- | ---------- | ------------------------------------- |
| `⚠ 重复`   | 多路径安装 | PATH 中发现 ≥2 条去重路径             |
| `⚠ 过期`   | 版本落后   | 本地 semver < 最新 semver             |
| `✓ 正常`   | 无需处理   | 已安装且版本不落后                    |
| `— 不适用` | 平台不匹配 | 仅限 macOS 的工具在 Windows 上运行    |
| `✗ 缺失`   | 未安装     | PATH 中未找到该工具                   |

## 行排序规则

表格行按状态分组排序，组内保持注册顺序：

| 优先级 | 组别     | 行数规则                                                  |
| ------ | -------- | --------------------------------------------------------- |
| 1      | ⚠ 重复   | 每个安装路径独占一行（首行显示工具名+序号，后续行显示 ↳） |
| 2      | ⚠ 过期   | 每工具 1 行                                               |
| 3      | ✓ 正常   | 每工具 1 行                                               |
| 4      | — 不适用 | 每工具 1 行                                               |
| 5      | ✗ 缺失   | 每工具 1 行                                               |

## 汇总行（紧跟表格）

```
---
扫描完成: 25 个工具
  ⚠ 重复: N  |  ⚠ 过期: N  |  ✓ 正常: N  |  — 不适用: N  |  ✗ 缺失: N
```

## 配置审计（仅控制台输出，不写入报告文件）

| 平台               | 标题                                | 审计范围                      |
| ------------------ | ----------------------------------- | ----------------------------- |
| Windows (PS1)      | `本地配置审计 (PATH / Environment)` | PATH 条目 + 环境变量          |
| macOS/Linux (bash) | `本地配置审计 (Shell RC / PATH)`    | Shell RC 文件内容 + PATH 模式 |

审计内容：

- 硬编码版本路径（Python/Node/Go/Java/JDK/.NET/Rust/Flutter/PHP）
- 环境变量中的过期值（JAVA_HOME / GOROOT / GOPATH）
- Windows 特有：PATH 中不存在的目录
- macOS/Linux 特有：Shell RC 文件中的旧版 Homebrew 路径、已弃用 conda activate 写法
- 无问题时输出：`✅ 未发现可疑的硬编码版本路径或过期配置`
